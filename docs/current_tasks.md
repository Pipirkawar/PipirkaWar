## 🛑 Известные блокеры / открытые вопросы PR-а

- **Phase-3 multi-membership ограничение** (наследие 3.6-A). В Phase 3 игрок может состоять максимум в **одном** клане (`UNIQUE` constraint на `clan_members.player_id` ещё с миграции `0006_clan_members`). Поэтому `count_active_for_player` фактически возвращает `0` или `1`, и `tribe_bonus_cm ∈ {0, cm_per_tribe}`. Интерфейс готов к Phase 4+ (multi-membership), но реальный `+131 см cap` будет достигаться только после снятия `UNIQUE`-constraint и реализации UI «вступить в племя × N» (вне скоупа Спринта 4.1).
- **Опциональный hint `oracle-no-tribes-hint`** (3 нулевых `/predict` подряд → «*вступай в новые племена и получай больше см*») — **не реализован** в 3.6-B. Помечен как feature-flag `oracle.show_no_tribes_hint` (default `false`). Реальная реализация требует трекинга последних 3 invocations — отдельный enhancement.
- **`ClaimPrize` — выплата зарезервированных лотов — задача 4.1-D.** В 4.1-C реализован picker `crypto_lot` (выбор активного лота при сработке) + резервирование `ACTIVE → RESERVED` + race-fallback на LengthGain. Но реальная выплата TON/USDT-лотов отсутствует — лоты висят в статусе `RESERVED` неопределённо долго. 4.1-D добавит use-case `ClaimPrize(player, lot_id, recipient_address)` + handler «Привязать кошелёк» (TON Connect) + транзакцию через TON SDK; refund-таймаут (`RESERVED → ACTIVE` через `lot_ttl_seconds`-cron + audit `PRIZE_LOT_REFUNDED`); жёсткая защита `actual_fee > fee_buffer` → refund-в-пул.
- **`InMemoryFeeEstimator` — заменяется в 4.1-D.** В 4.1-C использован константный `IFeeEstimator`-адаптер (STARS=0, TON_NANO=10_000_000 ≈ 0.01 TON, USDT_DECIMAL=200_000 ≈ 0.2 USDT-decimal — буфер на TON-газ jetton-перевода). На реальном TON sandbox-е fee-буфер должен опираться на P95 за 7 дней через `TonRpcFeeEstimator` (заменит `InMemoryFeeEstimator` через тот же `IFeeEstimator`-порт в `bot/main.py::Container`).
- **`ORM.__table_args__`-CHECK дрейф (см. C.6.e).** `AuditLogORM.__table_args__` `audit_log_source_whitelist`-CHECK устарел относительно Alembic-миграций; пере-синхронизирован вручную в C.6.e.
